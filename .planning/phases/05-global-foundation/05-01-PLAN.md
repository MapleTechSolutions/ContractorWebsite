  <action>
Make the following four targeted edits to globals.css. Read the file first to confirm current line numbers, then apply each change.

**Change 1 — :root block (line ~5): add -webkit-tap-highlight-color**

Current :root block ends at line 17 with `}`. Add one line before the closing brace:
```css
  -webkit-tap-highlight-color: transparent;     /* MFND-04 */
```

**Change 2 — body block (lines 36-47): add three properties, update one**

Current body block:
```css
body {
  font-family: var(--font-montserrat), system-ui, sans-serif;
  color: #ffffff;
  background: #0f1f2d;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
  overflow-x: hidden;
  text-rendering: optimizeSpeed;
  -webkit-tap-highlight-color: rgba(245, 166, 35, 0.2);
}
```

Replace with (changes: add font-size, line-height, touch-action; change tap-highlight to transparent):
```css
body {
  font-family: var(--font-montserrat), system-ui, sans-serif;
  font-size: 16px;                              /* MTYP-01 — prevents iOS zoom on input focus */
  line-height: 1.5;                             /* MTYP-02 — readable body text baseline */
  color: #ffffff;
  background: #0f1f2d;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
  overflow-x: hidden;
  text-rendering: optimizeSpeed;
  touch-action: manipulation;                   /* MFND-03 — eliminates 300ms tap delay on body */
  -webkit-tap-highlight-color: transparent;     /* MFND-04 — no amber flash on tap (mobile UX) */
}
```

NOTE: The amber tap highlight (`rgba(245, 166, 35, 0.2)`) was intentional brand styling but is being replaced with `transparent` as required by MFND-04. This is an intentional design change.

**Change 3 — Remove redundant input font-size media query**

Remove the entire block at lines ~49-54:
```css
/* Prevent zoom on input focus (iOS) */
@media screen and (max-width: 768px) {
  input, select, textarea {
    font-size: 16px !important;
  }
}
```

This block is now redundant — `font-size: 16px` on body sets the base, and `text-base` on `.form-input` class explicitly uses `text-base`. The `!important` is no longer needed.

**Change 4 — Add touch-action on all interactive elements (MPERF-01)**

After the body block (before the `/* Custom scrollbar */` comment), add:
```css
/* Eliminate 300ms tap delay on all interactive elements */
a, button {
  touch-action: manipulation;
}
```

The `btn-*` utility classes already include Tailwind's `touch-manipulation` — this new rule is consistent with those, not a conflict.
