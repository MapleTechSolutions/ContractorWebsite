**src/components/SeasonalBar.tsx — Create as Client Component:**

```tsx
"use client"

import { useState, useEffect } from "react"
import Link from "next/link"
import { seasonalMessage } from "@/data/seasonal"

const DISMISS_KEY = "seasonal-bar-dismissed"

export default function SeasonalBar() {
  const [visible, setVisible] = useState(false)

  useEffect(() => {
    if (!seasonalMessage.active) return
    const dismissed = sessionStorage.getItem(DISMISS_KEY)
    if (!dismissed) {
      setVisible(true)
    }
  }, [])

  function handleDismiss() {
    sessionStorage.setItem(DISMISS_KEY, "true")
    setVisible(false)
  }

  if (!visible) return null

  return (
    <div className="sticky top-0 z-[60] bg-[#F5A623] text-[#0f1f2d]">
      <div className="container-custom py-2 flex items-center justify-between gap-4">
        <div className="flex items-center gap-3 flex-1 min-w-0">
          <span className="text-sm font-semibold truncate">
            {seasonalMessage.text}
          </span>
          <Link
            href={seasonalMessage.ctaHref}
            className="flex-shrink-0 text-xs font-bold uppercase tracking-wider bg-[#0f1f2d] text-[#F5A623] px-3 py-1 rounded-full hover:bg-[#2d4f64] transition-colors"
          >
            {seasonalMessage.ctaText}
          </Link>
        </div>
        <button
          onClick={handleDismiss}
          aria-label="Dismiss announcement"
          className="flex-shrink-0 w-7 h-7 flex items-center justify-center rounded-full hover:bg-[#0f1f2d]/10 transition-colors touch-manipulation"
        >
          <svg className="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
            <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M6 18L18 6M6 6l12 12" />
          </svg>
        </button>
      </div>
    </div>
  )
}
```

**Key implementation notes:**
- `sticky top-0 z-[60]` ensures the bar scrolls away naturally. The fixed Header at `z-50` stays below it while bar is visible.
- `useEffect` reads sessionStorage only on the client (avoids SSR hydration mismatch). State starts `false` (hidden) and is set to `true` only after the client-side check confirms not dismissed.
- `if (!visible) return null` means the component mounts but renders nothing — no layout shift issues with SSR.
- `container-custom` keeps the bar aligned with the rest of the site layout.

## Notes

**SeasonalBar sticky vs fixed note:** The bar uses `sticky top-0 z-[60]`. The Header uses `fixed top-0 z-50`. When the page loads, the sticky bar is in normal flow above the fixed header — both show. As the user scrolls, the sticky bar scrolls off the top, and the fixed header stays. This is the intended UX: bar is visible on load, disappears on scroll, header always stays. No JavaScript or CSS variable offsets needed.

**Hero.tsx has TWO forms:** One in the desktop layout and one in the mobile layout (they are separate JSX blocks, not a single shared form). Both must have `onSubmit={handleSubmit}` added and both must show the loading/success states. A single shared `formStatus` state variable manages both (they don't both show at the same time due to responsive breakpoints).

**Contact.tsx submit handler is already correct** — it uses `await new Promise(resolve => setTimeout(resolve, 1000))` which provides 1 second of loading state before showing success. This is the correct implementation per the project requirement (no real backend, just proper UI state). Do NOT replace this with a Formspree endpoint or any network call.

**Form field values (formData state) should be reset on success** — Contact.tsx already does this: `setFormData({ name: "", email: "", phone: "", service: "", message: "" })`. Keep that.
