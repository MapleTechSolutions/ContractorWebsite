# Plan 01-01: Color, Font, Logo, Layout

## Goal

The visual foundation is set: brand color tokens live in tailwind.config.ts, all legacy dark green hex values are purged from globals.css, Montserrat loads via next/font/google, the Big Country logo appears in header and footer, and the SeasonalBar slot is wired into layout.tsx.

## Context

The codebase is a generic contractor template using dark green (#094026) and gold (#FCB215) throughout. The Big Country logo SVG exists at the project root but has never been copied to `abc-roofing/public/`. Header and Footer contain inline SVG house icons. Font loading uses a render-blocking `@import` in globals.css. This plan establishes everything else in Phase 1 builds on.

## Notes

**Execution order is strict within this plan:** Task 1 (copy logo + CSS foundation) must complete before Task 3 (logo integration into components). Task 2 (layout.tsx) can happen after Task 1 in parallel with Task 3 if desired, but the logo file must exist first.

**SeasonalBar import in layout.tsx:** SeasonalBar.tsx does not exist yet — it is created in Plan 01-03. If TypeScript compilation fails due to the missing import, comment out the `<SeasonalBar />` line with a `// TODO: Plan 01-03` note. The slot placement in the DOM is what matters here.

**next/image and SVG:** `<Image src="/logo.svg">` with explicit `width` and `height` is correct. SVG files in `public/` are served as static files by Next.js with no optimization needed. The `unoptimized` prop is NOT required for SVGs.

**tailwind.config.ts is Tailwind v3:** Use `extend.colors` (standard Tailwind v3 pattern), NOT `@theme {}` (that is Tailwind v4 syntax). The research confirms this is a v3 project.
